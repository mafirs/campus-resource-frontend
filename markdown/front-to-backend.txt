校级活动场地与物资管理系统 - 后端接口文档

========================================

项目概述
----------------------------------------
项目名称：校级活动场地与物资管理系统
后端框架：Flask + Flask-RESTful + Flask-SQLAlchemy
数据库：MySQL (暂缓实现)
认证方式：JWT Token
接口风格：RESTful API

========================================

认证机制
----------------------------------------

JWT Token 规范
{
  "userId": 1,
  "username": "admin",
  "role": "admin",
  "iat": 1640995200,
  "exp": 1641081600
}

请求头格式
Authorization: Bearer <token>
Content-Type: application/json

统一响应格式
{
  "code": 200,
  "message": "success",
  "data": {}
}

认证模块 API
----------------------------------------

1. 用户登录
接口地址: POST /api/auth/login

请求参数:
{
  "username": "string",
  "password": "string"
}

返回数据:
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": 1,
      "username": "admin",
      "realName": "管理员",
      "role": "admin",
      "email": "admin@example.com"
    }
  }
}

业务逻辑:
- 验证用户名密码
- 生成JWT Token
- 返回用户信息和Token
- Mock数据: admin→admin, reviewer→reviewer, 其他→user

2. 获取当前用户信息
接口地址: GET /api/auth/profile
请求头: Authorization: Bearer <token>

返回数据:
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "username": "admin",
    "realName": "管理员",
    "email": "admin@example.com",
    "role": "admin"
  }
}

3. 用户登出
接口地址: POST /api/auth/logout
请求头: Authorization: Bearer <token>

返回数据:
{
  "code": 200,
  "message": "登出成功",
  "data": null
}

申请管理 API
----------------------------------------

1. 创建申请
接口地址: POST /api/applications
权限要求: user, admin, reviewer

请求参数:
{
  "activityName": "string",
  "activityDescription": "string",
  "venueId": 1,
  "startTime": "2024-01-01T14:00:00Z",
  "endTime": "2024-01-01T16:00:00Z",
  "materials": [
    {
      "materialId": 1,
      "quantity": 2
    },
    {
      "materialId": 2,
      "quantity": 10
    }
  ]
}

返回数据:
{
  "code": 200,
  "message": "申请提交成功",
  "data": {
    "id": 1,
    "activityName": "社团招新活动",
    "status": "pending",
    "createdAt": "2024-01-01T14:00:00Z"
  }
}

业务逻辑:
- 验证场地在指定时间段是否可用
- 验证物资库存是否充足
- 减少对应物资的可用数量
- 创建申请记录和物资使用记录

权限要求: reviewer, admin

查询参数:
- page: 页码 (默认1)
- size: 每页数量 (默认10)

返回数据:
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "activityName": "社团招新活动",

        "venueLocation": "A栋101",
        "venueCapacity": 200,
        "startTime": "2024-01-01T14:00:00Z",
        "endTime": "2024-01-01T16:00:00Z",
        "materials": [
          {
            "materialId": 1,
            "materialName": "投影仪",
            "requestedQuantity": 1,

            "stockStatus": "sufficient"

          {

- sufficient: 库存充足
- low: 库存紧张
- insufficient: 库存不足

2. 审批申请 - 通过
接口地址: PUT /api/applications/<int:application_id>/approve
权限要求: reviewer, admin

返回数据:
{
  "code": 200,
  "message": "申请已通过",
  "data": {
    "id": 1,
    "status": "approved",
    "reviewerId": 3,
    "reviewedAt": "2024-01-01T15:00:00Z"
  }
}

3. 审批申请 - 驳回
接口地址: PUT /api/applications/<int:application_id>/reject
权限要求: reviewer, admin

请求参数:
{
  "reason": "string"  // 驳回理由 (必填)
}

返回数据:
{
  "code": 200,
  "message": "申请已驳回",
  "data": {
    "id": 1,
    "status": "rejected",
    "rejectionReason": "时间冲突",
    "reviewerId": 3,
    "reviewedAt": "2024-01-01T15:00:00Z"
  }
}

业务逻辑:
- 驳回申请时，归还对应的物资库存
- 更新申请状态为rejected
- 记录驳回理由和审批人

重要业务规则
----------------------------------------

申请规则
1. 同一场地同一时间不能有多个approved状态的申请
2. 物资申请数量不能超过当前可用数量
3. 申请时间不能是过去时间
4. 结束时间必须大于开始时间
5. 只有pending或approved状态的申请可以取消

审批规则
1. 只有pending状态的申请可以审批
2. 驳回申请必须填写理由
3. 审批通过后，物资库存会被占用
4. 审批驳回后，物资库存会释放

库存规则
1. 物资总数量 >= 已占用数量 + 可用数量
2. 申请通过时，减少可用数量
3. 申请取消或驳回时，增加可用数量
4. 管理员可以手动调整物资总数量

# app/__init__.py
from flask import Flask
from flask_cors import CORS
from flask_jwt_extended import JWTManager

def create_app():
    app = Flask(__name__)
    app.config.from_object('config.Config')

    # 初始化扩展
    CORS(app)
    jwt = JWTManager(app)

    # 注册蓝图
    from app.api import auth_bp, venues_bp, materials_bp, applications_bp, approvals_bp, dashboard_bp

    app.register_blueprint(auth_bp, url_prefix='/api/auth')
    app.register_blueprint(venues_bp, url_prefix='/api/venues')
    app.register_blueprint(materials_bp, url_prefix='/api/materials')
    app.register_blueprint(applications_bp, url_prefix='/api/applications')
    app.register_blueprint(approvals_bp, url_prefix='/api/approvals')
    app.register_blueprint(dashboard_bp, url_prefix='/api/dashboard')

    return app

3. 错误处理
# utils/response.py
from flask import jsonify

def success_response(data=None, message="success"):
    return jsonify({
        "code": 200,
        "message": message,
        "data": data
    })

def error_response(code, message, data=None):
    return jsonify({
        "code": code,
        "message": message,
        "data": data
    }), 400 if code < 500 else 500
