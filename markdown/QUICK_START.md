# 🚀 快速开始指南

## 第一步: 进入项目目录
```bash
cd venue-management-system
```

## 第二步: 启动开发服务器
```bash
npm run dev
```

你会看到类似这样的输出:
```
  VITE v7.1.7  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

## 第三步: 打开浏览器
访问: http://localhost:5173

## 第四步: 测试不同角色

### 测试管理员 (Admin)
1. 用户名: `admin`
2. 密码: 随便输入（例如: `123456`）
3. 点击"登录"
4. 观察侧边栏菜单，你会看到:
   - ✅ 首页
   - ✅ 场地管理
   - ✅ 物资管理

### 测试审核员 (Reviewer)
1. 退出登录（点击右上角头像 → 退出登录）
2. 用户名: `reviewer`
3. 密码: 随便输入
4. 点击"登录"
5. 观察侧边栏菜单，你会看到:
   - ✅ 首页
   - ✅ 待我审批

### 测试普通用户 (User)
1. 退出登录
2. 用户名: 随便输入（例如: `zhangsan`）
3. 密码: 随便输入
4. 点击"登录"
5. 观察侧边栏菜单，你会看到:
   - ✅ 首页
   - ✅ 发起申请

## 第五步: 测试权限控制

### 测试 1: 尝试访问无权限的页面
1. 以普通用户 (user) 身份登录
2. 在浏览器地址栏输入: `http://localhost:5173/manage/venues`
3. 按回车
4. 你会被自动重定向到 403 页面（无权限）

### 测试 2: 未登录访问
1. 退出登录
2. 在浏览器地址栏输入: `http://localhost:5173/dashboard`
3. 按回车
4. 你会被自动重定向到登录页

### 测试 3: 刷新页面保持登录
1. 以任意角色登录
2. 按 F5 刷新页面
3. 你会发现仍然保持登录状态（不需要重新登录）

## 🐛 常见问题

### Q: 登录后看不到菜单？
A: 检查浏览器控制台是否有错误。确保使用了正确的用户名（admin/reviewer/其他）。

### Q: 刷新页面后退出登录了？
A: 检查浏览器是否禁用了 localStorage。本项目使用 localStorage 存储登录状态。

### Q: Element Plus 组件没有样式？
A: 确保 main.js 中导入了 `element-plus/dist/index.css`。

### Q: 路由跳转不工作？
A: 检查路由配置是否正确，确保 main.js 中正确 use(router)。
