## 👥 三个角色完整实现

### 1. Admin (系统管理员)
**权限**: 最高权限，管理基础数据

**可访问页面**:
- 首页
- 场地管理 (CRUD)
- 物资管理 (CRUD)

**核心功能**:
- 管理所有场地（新增、编辑、删除、状态切换）
- 管理所有物资（新增、编辑、删除、分类、库存）
- 数据实时同步到其他模块

### 2. User (普通用户)
**权限**: 发起申请，查看自己的申请

**可访问页面**:
- 首页
- 发起申请
- 我的申请

**核心功能**:
- 发起场地和物资的申请
- 动态添加/删除物资项
- 查看自己的所有申请
- 查看申请状态（待审核/已通过/未通过/已取消）
- 取消待审核或已通过的申请

### 3. Reviewer (审核员)
**权限**: 审批用户提交的申请

**可访问页面**:
- 首页
- 待我审批

**核心功能**:
- 查看所有待审批的申请
- 展开查看申请详情（时间、物资、库存状态）
- 通过申请
- 驳回申请（必须填写理由）
- 智能库存检查辅助决策

## 🔄 完整业务流程

```
┌─────────────────────────────────────────────────────────┐
│                    业务流程闭环                          │
└─────────────────────────────────────────────────────────┘

1. Admin 管理基础数据
   ├─ 创建场地（大学生活动中心、体育馆...）
   └─ 创建物资（折叠椅、投影仪...）
         ↓
2. User 发起申请
   ├─ 选择场地（只能选开放预约的）
   ├─ 选择时间
   ├─ 动态添加物资
   └─ 提交申请（状态：待审核）
         ↓
3. Reviewer 审批申请
   ├─ 查看待审批列表
   ├─ 展开查看详情
   ├─ 检查库存状态
   └─ 做出决定
       ├─ 通过 → 状态：已通过
       └─ 驳回 → 状态：未通过（附理由）
         ↓
4. User 查看结果
   ├─ 在"我的申请"看到状态更新
   ├─ 如果已通过：可以取消
   └─ 如果未通过：查看驳回理由
```
