Claude Code 斜杠命令完整指南
============================================

 Claude Code 是一个强大的 AI 编程助手，当你在聊天框中输入 "/" 时，
会显示各种可用的命令和功能。本指南详细介绍这些功能及其日常使用。

============================================
📋 斜杠命令总览
============================================

在 Claude Code 聊天框中输入 "/" 可以看到以下主要命令类别：

1. 📁 文件操作命令
2. 🔍 搜索和分析命令
3. 🛠️ 代码操作命令

============================================
📁 文件操作命令 (高频使用)
============================================

/new <filename> - 创建新文件
---------------------------------------------
用途：快速创建新文件并添加初始内容
使用频率：⭐⭐⭐⭐⭐
示例：
• /new app.py - 创建 Python 文件
• /new README.md - 创建 Markdown 文件
• /new styles.css - 创建 CSS 文件

/open <filename> - 打开文件
---------------------------------------------
用途：打开并查看现有文件内容
使用频率：⭐⭐⭐⭐⭐
示例：
• /open package.json - 查看项目配置
• /open src/main.js - 查看主文件

/edit <filename> - 编辑文件
---------------------------------------------
用途：编辑指定文件的内容
使用频率：⭐⭐⭐⭐⭐
示例：
• /edit config.json - 修改配置文件
• /edit index.html - 编辑 HTML 文件

/delete <filename> - 删除文件
---------------------------------------------
用途：删除不需要的文件
使用频率：⭐⭐⭐
示例：
• /delete old-file.js - 删除旧文件

============================================
🔍 搜索和分析命令 (高频使用)
============================================

/find <pattern> - 查找文件
---------------------------------------------
用途：在项目中查找匹配模式的文件
使用频率：⭐⭐⭐⭐⭐
示例：
• /find "*.js" - 查找所有 JS 文件
• /find "*test*" - 查找包含 test 的文件
• /find "src/**/*.vue" - 查找 Vue 组件

/search <text> - 搜索文本内容
---------------------------------------------
用途：在文件内容中搜索指定文本
使用频率：⭐⭐⭐⭐⭐
示例：
• /search "function login" - 搜索登录函数
• /search "TODO" - 查找待办事项
• /search "error" - 搜索错误相关代码

/analyze <file/directory> - 分析代码
---------------------------------------------
用途：分析文件或目录的代码结构
使用频率：⭐⭐⭐⭐
示例：
• /analyze src/ - 分析源码目录
• /analyze main.py - 分析主程序
• /analyze . - 分析整个项目

============================================
💡 实用技巧和组合用法
============================================

🔍 搜索技巧
---------------------------------------------
• /find "*.js" | /search "function" - 找到所有 JS 文件并搜索函数
• /search "TODO" | /explain --all - 查找所有待办事项并解释

🛠️ 开发流程
---------------------------------------------
1. /analyze . - 分析整个项目
2. /find "*.py" | /review --security - 审查所有 Python 文件
3. /test --all | /debug - 运行测试并调试问题
4. /refactor main.py | /improve --performance - 重构并优化性能

📝 文档生成
---------------------------------------------
• /structure > docs/structure.md - 保存项目结构
• /summary > README.md - 生成项目简介
• /docs API > docs/api.md - 生成 API 文档

============================================
❓ 常见问题解答
============================================

Q: 如何查看所有可用命令？
A: 直接输入 "/" 然后按 Tab 键，或使用 /help

Q: 命令太多记不住怎么办？
A: 重点关注高频使用的 5 个命令：/explain, /improve, /find, /search, /debug

Q: 如何组合使用多个命令？
A: 使用管道符号 "|" 连接命令，如：/find "*.js" | /search "function"

Q: 命令执行失败了怎么办？
A: 检查文件路径是否正确，权限是否足够，或使用 /debug 调试

Q: 如何撤销命令的操作？
A: 大部分文件操作可以通过 /edit 撤销，或使用版本控制工具回退
